# 굴착 작업계획서 Excel 생성 가능성 판단 — 6단계

**작성일**: 2026-04-24

---

## 1. Excel 단일 시트 가능 여부

| 항목 | 판단 | 근거 |
|------|------|------|
| 단일 시트 출력 | **가능** | 제목+메타+법정항목+반복행+서명 = 29행 (A4 1~2페이지) |
| A4 가로 맞춤 | **가능** | 8열, fitToWidth=1, 열 너비 합계 94 units |
| 세로 페이지 수 | 1~2페이지 | 기본 29행 ≈ 720pt, A4 세로 ≈ 729pt 경계 |
| 복합 시트 필요 여부 | **없음** | 굴착 단순 서식 — 단일 시트로 충분 |

---

## 2. 반복 행 구조 적합성

| 항목 | 판단 |
|------|------|
| 반복 행 타입 | 작업단계별 안전조치 (step) |
| 기본 행 수 | 10행 고정 (기본), 초과 시 동적 추가 |
| 최소 행 | 0 (steps 없으면 10행 공란) |
| 반복 행 최대 | 제한 없음 (단, 20행 초과 시 2페이지 분할) |
| 구현 패턴 | 교육일지 `attendee` 반복행과 동일 패턴 — **검증된 구조** |

---

## 3. 자동화 가능한 필드 비율

| 구분 | 필드 수 | 비율 |
|------|---------|------|
| AUTO (자동 채번·날짜) | 2 (`step_no` 채번, `prepared_date` 선택적 오늘날짜) | 9% |
| USER 필수 (법정) | 7 | 32% |
| USER 관행 | 8 | 36% |
| USER 반복 (step 3필드 × n행) | n×3 | — |
| 강제 공란 | 2 (서명란) | 9% |

> **자동화 기여도 낮음** — form_generation_lock.md 기 판단과 일치. 서식 생성 자체가 가치 (수기 → 구조화 출력).

---

## 4. 기술적 구현 난이도

| 항목 | 난이도 | 이유 |
|------|-------|------|
| openpyxl 구현 | **낮음** | 교육일지 builder와 동일 패턴 |
| 병합셀 구조 | **낮음** | 병합 범위 명확히 정의됨 |
| 동적 step 확장 | **낮음** | list 순회 + row 오프셋 계산 |
| 법정 항목 라벨 wrap | **낮음** | wrap_text=True + 고정 높이 |
| 인쇄 설정 | **낮음** | 교육일지와 동일 |
| 도면 영역 | **제외** | Excel 범위 외 (수기) |
