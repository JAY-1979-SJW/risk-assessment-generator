## 대량 생성 전략

### 전략 1: evidence 공유 묶음 처리

동일 법령 조항을 공유하는 문서들을 한 번에 처리:
- 밀폐공간 묶음: CL-010 → PTW-001 → WP-014 (동일 조항 재사용)
- 차량계 묶음: WP-008/WP-009 evidence 완성 → EQ-001/EQ-002/EQ-004 공유
- 크레인 묶음: WP-006/WP-007 evidence 완성 → EQ-003/EQ-004 공유

### 전략 2: 공통 템플릿 적용

CM/EM/DL/SP 카테고리는 구조가 단순하고 법령 의무 사항이 적음:
- 기존 `electrical_facility_checklist_builder.py` 패턴 재사용
- 서식 섹션 수 최소화 (3~5섹션)
- 증거 없이 practical 서식으로 일괄 처리

### 전략 3: EQ 시리즈 일괄 구현

EQ-005~EQ-016 (12종)은 모두 장비 특화 작업계획서:
- 공통 구조: 기본정보 + 장비사양 + 점검항목 + 안전조치 + 서명
- 기존 WP 계획서 builder를 장비 이름만 변경하여 재사용 가능
- evidence는 산안규칙 해당 장비 조항 (각 2~3 조항) PARTIAL_VERIFIED 처리

### 전략 4: 고우선순위 단독 구현

PPE-001, PTW-005, PTW-008은 패키지 완성에 직결:
- 별도 설계 검토 후 개별 구현
- PPE-001: 전 패키지 advisory 해소 → 완성도 지표 개선
