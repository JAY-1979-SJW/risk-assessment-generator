# PTW-007 중량물 인양·중장비사용 작업 허가서 — 구현 설계 문서 v1.0

작성일: 2026-04-25  
버전: v1.0  
상태: DONE (builder v1.0 구현 완료, evidence NEEDS_VERIFICATION)

---

## 1. 목적

중량물 인양 작업 및 크레인·굴착기 등 중장비사용 작업 당일,  
작업 전 인양위험 및 중장비 사용위험 확인과 안전조치 이행 여부를 기록하고  
작업허가자로부터 공식 승인을 받는 절차를 표준화한다.

KOSHA GUIDE P-94-2021의 '보충작업허가(Supplementary Work Permit)' 중  
**중장비사용작업허가** 구조를 참고하여 중량물 인양 허가와 중장비사용 허가 기능을 통합한다.

---

## 2. 적용 범위

- 이동식 크레인·카고크레인·체인블록·레버블록·지게차 보조 등 양중기를 이용한 중량물 인양 작업
- 굴착기 인양작업 특칙(제221조의5) 해당 작업
- 원청 안전관리규정에서 허가를 요구하는 중량물 취급 작업

---

## 3. 법정 별지 서식 없음

산업안전보건기준에 관한 규칙에 중량물 인양 작업 허가서 법정 별지 서식이 존재하지 않는다.  
본 서식은 관련 법령 의무 이행 확인 및 원청 안전관리 요구를 충족하기 위한 **자체 표준서식**이다.

- `legal_status`: `practical`
- `evidence_status`: `NEEDS_VERIFICATION`

## 9. KOSHA P-94-2021 중장비사용작업허가와의 관계

KOSHA GUIDE P-94-2021은 안전작업허가 체계 내에서 '보충작업허가(Supplementary Work Permit)'의 하나로 **중장비사용작업허가**를 규정한다.

| 구분 | 내용 |
|------|------|
| 허가 유형 | 보충작업허가 (중장비사용작업허가) |
| 주요 확인 항목 | 중장비 종류·규격, 작업반경·경로, 신호수 배치, 지반·아웃트리거, 출입통제, 정격하중 |
| PTW-007 반영 | 섹션 5(양중기·정격하중), 섹션 8(지반·아웃트리거), 섹션 9(신호수·출입통제) |
| evidence 상태 | NEEDS_VERIFICATION (PTW-007-K3 — PDF 원문 미수집) |

PTW-007은 중량물 인양 허가(법령 기반)와 중장비사용작업허가(KOSHA 권고 기반)를 단일 서식으로 통합하여, 현장에서 두 허가를 별도 발행하는 부담을 줄인다.

## 10. WP-005 중량물 취급 작업계획서와의 관계

| 서식 | 역할 |
|------|------|
| **WP-005** (TODO) | 법정 중량물 취급 작업계획서 (제38조 ① 11호) — 사전 작성 의무 |
| **PTW-007** | 인양 허가 당일 확인서 — 당일 안전조치 이행 기록 |

- **PTW-007은 WP-005를 대체하지 않는다.**
- 작업계획서(WP-005) 선행 작성 후 PTW-007 발행이 원칙
- builder 내 고정 문구로 명시: "본 허가서는 WP-005 중량물 취급 작업계획서를 대체하지 않는다."

## 10. CL-003 장비점검표와의 관계

- 이동식 크레인·카고크레인 등 장비 사용 시 CL-003 건설장비 일일 사전점검표를 별도로 확인해야 한다.
- PTW-007 섹션 3(작업계획서 확인)에 "CL-003 건설장비 일일점검표 확인" 항목 포함
- builder 내 고정 문구: "이동식크레인 등 양중기 사용 시 해당 작업계획서, 장비사용계획서, 장비점검표를 별도로 확인해야 한다."

## 12. RA-001 위험성평가와의 관계

- FIRE_PIPE_INSTALL, MECH_PIPE_INSTALL, COMMON_HEAVY_LIFTING 등 공종 프리셋의 required_permits에 PTW-007이 이미 등록되어 있다.
- RA-001 위험성평가표 생성 시 관련 허가서 항목으로 PTW-007이 자동 반영된다.
- RA-001 adapter, risk_assessment_adapter.py는 수정하지 않음 — 공종 프리셋 연결로 자동 처리된다.

---

## 13. RA-004 TBM과의 관계

- TBM 안전점검 일지(RA-004)의 permit_check 필드에 PTW-007이 공종 프리셋 연결로 자동 반영된다.
- tbm_log_adapter.py는 수정하지 않음.
