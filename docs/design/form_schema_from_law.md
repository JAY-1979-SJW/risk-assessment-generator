# 법령 기반 필드 스키마 정의 v1

**작성일**: 2026-04-24
**근거**: `docs/design/form_requirements_spec.md`
**원칙**: 법정 항목과 실무 관행 항목을 명확히 구분. 자동 생성 가능 여부 표기.

필드 성격 코드:
- `[법정]` — 법령 조문에 명시된 항목
- `[관행]` — 법령 미명시, 실무 관행으로 기재
- `[AUTO]` — 엔진 또는 시스템이 자동 채울 수 있는 필드
- `[USER]` — 사용자 입력 필요
- `[REPEAT]` — 반복 행 구조 (테이블 행)


## 3. 공정안전보고서 스키마

### 3.1 구조 개요 (패키지 단위)

PSM은 단일 서식이 아닌 **4개 구성요소 × 개별 문서** 패키지.
Excel 단일 파일 구현 불가 → **구성요소별 개별 시트 또는 개별 파일** 로 분해.

```
PSM Package
├── PSM-1: 공정안전자료      (설비 목록 테이블 중심)
├── PSM-2: 공정위험성평가서  (위험 시나리오 테이블)
├── PSM-3: 안전운전계획      (8개 항목 체크리스트/서술)
└── PSM-4: 비상조치계획      (조직도 + 절차 서술)
```

## 4. 스키마 요약 — 자동화 가능성 평가

| 서식 | 총 필드 수 | [AUTO] 가능 수 | [USER] 필수 수 | 비고 |
|------|-----------|--------------|--------------|------|
| 작업계획서 (공통+굴착 기준) | 약 18 | 3 | 15 | 작업유형별 항목 다름 |
| 교육일지 | 약 14 (메타) + 수강자 행 | 3 | 11 | 수강자 행은 건수 제한 없음 |
| PSM-1 (공정안전자료) | 약 12 (메타) + 반복 행 | 2 | 10 | 도면 첨부 별도 |
| PSM-2 (위험성평가서) | 약 10 (시나리오 행) | 1 | 9 | HAZOP 등 전문 분석 전제 |
| PSM-3·4 | 체크리스트 구조 | 0 | 전부 | 서술형 문서 |

**판정**: 작업계획서·교육일지는 Excel 단일 파일 생성 가능. PSM은 패키지 설계 필요.
