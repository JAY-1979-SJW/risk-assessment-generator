### 2.3 KOSHA OPS 스크래핑 (`scraper/kosha_files/`)

**수집 방식**: KOSHA 지식DB(kosha.or.kr) OPS 카테고리 스크래핑.
**폴더 구조**: `건설업/`, `제조업/`, `기타산업/`, `image_samples/` × `OPS/`, `기타/` × 날짜 폴더 × 파일들.
**총 바이너리**: 2,382 PDF + 400 zip + 87 jpg + 52 bin + 31 mp4 + 10 png + 5 hwp + 1 pptx.

**성격**
- 압도적 다수는 **교재/안내서/포스터/사례집/OPL 1페이지 요약지** — 편집형 서식 템플릿이 **아니다**.
- 따라서 **바이너리의 95% 이상은 "참고자료(B)"** 로 분류되며, "서식(A/B 템플릿)"으로 직접 쓰이는 것은 제한적.
- hwp 5건 중 편집형 양식 여부는 Step 3 실측 필요.

**결론**
- 본 인벤토리에서는 **출처 단위로만 등재**하고, 파일 단위 분류는 Step 4에서 "서식/비서식" 1차 필터 후 서식만 매트릭스 진입.

## 5. 인벤토리 컬럼 스키마 (Step 3~4 공통)

Step 2 저장 구조 확정 이후, Step 3 수집 로그와 Step 4 권위 매트릭스는 다음 컬럼을 공유한다.

| 컬럼 | 설명 |
|------|------|
| `id` | 내부 일련번호 (출처코드-순번, 예: `LIC-001`, `MOEL-042`, `KOSHA-217`, `INT-003`) |
| `source` | 출처 (`law.go.kr/licbyl`, `moel.go.kr`, `kosha.or.kr`, `internal`) |
| `source_id` | 출처 고유 ID (licbyl lawId, MOEL 게시물 ID, KOSHA 문서 ID) |
| `source_url` | 원본 URL (있으면) |
| `doc_title` | 문서명 |
| `doc_format` | 파일 형식 (hwp/hwpx/pdf/docx/xlsx/xls) |
| `doc_purpose` | 용도 (본표 / 점검표 / 계획서 / 규정 / 참고) |
| `authority_grade` | A / B / C |
| `is_original` | 원본 여부 (공공기관 직접 배포 = Y, 재배포/변형 = N) |
| `downloaded` | 실제 저장 여부 (Y/N) |
| `local_path` | 저장 경로 |
| `checksum` | SHA-256 (Step 3에서 기록) |
| `notes` | 비고 |
