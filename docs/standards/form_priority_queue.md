# 우선 확인 대상 큐 v1

**작성일**: 2026-04-23
**목적**: 안전서류 자동화 구현을 앞두고, 수집된 서식 원본 중 **1순위로 구조 분석·스키마 확정**에 들어갈 대상을 선정.
**근거**: `docs/standards/form_authority_matrix.md` v2, `data/forms/source_map.csv`

우선순위는 (1) **법적 의무**, (2) **위험성평가표 본표와의 연동 필수성**, (3) **현장 발생 빈도**, (4) **확보 여부·등급** 4축으로 결정.

---

## 1. 우선순위 티어 정의

| 티어 | 정의 | 다음 조치 |
|------|------|-----------|
| **P0** | 법적 의무 + 본 엔진 연동 필수 + **원본 확보 완료** | 즉시 구조 분석(셀 단위 실측) → 본표 출력 엔진 연동 |
| **P1** | 법적 의무이나 **원본 미확보** | Step 3 격차로 지정된 항목. 외부 수집 선행 후 P0 승격 |
| **P2** | 실무 보조 서식, 본 엔진 출력에 함께 나가면 좋은 부속 | 본표 안정화 이후 지원 |
| **P3** | 참고·교재·정책문서. 구현 대상 아님 | 참조자료로만 활용 |

## 6. 즉시 착수 순서 (권고)

1. **P0-1·P0-2 비교 분석** — 공문기반 16컬럼 vs KRAS 17컬럼. 현재 엔진 출력과의 컬럼 맵핑 확인 (이미 `form_schema_lock.md` 존재). 실측 갱신 필요.
2. **P0-4·P0-5 실측** — 유해위험방지계획서 hwp 열람 → 본표 출력에 "계획서 작성 필요 시 연동" 여부 결정.
3. **P1-1 외부 수집** — 작업계획서 원본 확보가 본 프로젝트의 가장 큰 블로커. `licbyl` API에 기준규칙 lawId 추가 수집 시도.
4. **P1-3·P1-4 외부 수집** — 교육일지·TBM 일지.
5. P2는 엔진 출력 안정화 이후 배치.

## 7. 큐 유지보수

- 본 큐는 `source_map.csv` 변경 시 수동 재작성. (구조가 단순하므로 자동화 부담 대비 효용 낮음.)

  - 원본 파일을 적절한 `data/forms/` 하위 폴더에 배치
  - `scripts/collect_forms_to_repository.py` 재실행 (또는 `INTERNAL_RULES`에 수동 등록 후 실행)
  - 본 큐에서 P1 → P0로 승격
